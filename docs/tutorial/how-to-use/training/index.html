    Model Training¶

    The haies model is a multi-head deep learning architecture (e.g., based on UNet or PVTv2) that simultaneously performs hedge segmentation and canopy height estimation.

    Training Strategy¶

    Training is divided into two phases for optimal stability:

    1. Phase 1: Pretraining: Training on height estimation only. This allows the model to learn the basic spatial structures and textures of the landscape before tackling the more challenging task of hedge segmentation.

    2. Phase 2: Full Training: Fine-tuning the model for both heights and hedge segmentation, starting from the weights learned in Phase 1.

    Training Subcommand¶

    The haies fit subcommand is used for both training phases.

    # General training command
haies fit \
    --config configs/reproducer/base.yaml \
    --config configs/reproducer/train.yaml \
    --model.init_args.net_ckpt_path="/path/to/checkpoints/last.ckpt"

    Configuration Files¶

    haies uses multiple YAML files to define the training environment.

    Reference Configurations

    Reference configurations for training are shown below. You can also find them in docs/example_configs/reproducer/.

    seed_everything: 41 # replace with true for a random seed

    Phase 1: Pretraining¶

    Train the model on the height target only.

    haies fit \
    --config configs/reproducer/base.yaml \
    --config configs/reproducer/pretrain.yaml

    Phase 2: Full Training¶

    After pretraining, train the full model. Ensure you provide the path to the best checkpoint from Phase 1 using --model.init_args.net_ckpt_path.

    haies fit \
    --config configs/reproducer/base.yaml \
    --config configs/reproducer/train.yaml \
    --model.init_args.net_ckpt_path="/path/to/pretrain/checkpoints/last.ckpt"

    Monitoring Training¶

    Progress and metrics are logged automatically. You can monitor them using TensorBoard:

    tensorboard --logdir /path/to/trainer/logs

    Look for the following metrics:

    • val/height/huber: Height estimation error (lower is better).

    • val/segment/f1: Segmentation quality (higher is better).
