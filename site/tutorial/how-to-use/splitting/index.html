    Dataset Splitting¶

    With the labels preprocessed, the next step is to divide your Area of Interest (AOI) into training, validation, and test sets. This ensures unbiased evaluation and prevents data leakage.

    Spatial Splitting¶

    haies uses a spatial split strategy. It divides the AOI into non-overlapping tiles, ensuring that training and evaluation data are geographically separated. A buffer zone is often used between sets to prevent spatial autocorrelation.

    Running the Splitting Tool¶

    The haies_create_split tool automates this process.

    # Create the dataset split
haies_create_split --config-path <path_to_config_dir> --config-name create_split

    Configuration Details¶

    The create_split.yaml configuration defines the parameters for the split:

    Reference Configuration

    The following configuration is used for dataset splitting. You can also find the file at docs/example_configs/splitting.yaml.

    # conf/config.yaml
_target_: haies.clis.create_split.Config

version: null  # Automatically set to timestamp if null

save_dir: ./outputs/splits
targets_data_dir:
  target_1: [./data/targets_1]
  target_2: [./data/targets_2]
bdortho_dir: [./data/bdortho]

test_side_length: 5_000
buffer: 1_000
test_area: 100_000_000
val_proportion: 0.1

seed: 42
overwrite: true
convert_to_tif: false

    Output Artifacts¶

    The tool generates several important files in your save_dir:

    1. geometries.parquet: Contains the boundaries and set assignment (train/val/test) for each tile.

    2. create_splits_config.yaml: The configuration used for the split.

    Class Imbalance and Loss Weighting¶

    Hedges typically occupy only a small fraction of the landscape. To address this class imbalance, you need to calculate the ratio of hedge pixels in your training set.

    Using gdalinfo -stats on the generated labels or analyzing the geometries.parquet file, you can determine the observed mean \(\mu\) of your labels. For a BCEWithLogitsLoss, the positive weight should be:

    \[pos\_weight = \frac{1}{\mu} - 1\]

    This value will be used in the Model Training configuration.

    Accelerating Training (Optional)¶

    haies_create_split defines the sets, but it does not create the data for training. If you want to accelerate training, you can convert the rasters into memory-mapped arrays using Memory Mapping. Otherwise, you can proceed directly to Model Training.
